多元梯度下降法处理技巧-特征缩放

  1. Feature Scaling 特征缩放

    • Idea: Make sure feature are on a similar scale. 确保每个特征的取值范围接近


    举例:

    • x1x_1x1​ = size ( 0 ~ 2000 feet2feet^2feet2 ); x2x_2x2​ = number of bedrooms ( 1 ~ 5 )
      这样损失函数会非常的狭长，收敛速度会非常慢
    • 应用特城缩放 x1=size(feet2)2000x_1 = \dfrac{size(feet^2)}{2000}x1​=2000size(feet2)​ , x2=numberofbedrooms5x_2 = \dfrac{number of bedrooms}{5}x2​=5numberofbedrooms​
      这样损失函数是个圆，梯度下降会更加快，收敛速度也会很快

    • Get every feature into approximately a −1⩽xi⩽1-1 \leqslant x_i \leqslant 1−1⩽xi​⩽1 range.
      将每个特征的取值转化到与 −1⩽xi⩽1-1 \leqslant x_i \leqslant 1−1⩽xi​⩽1 相近的范围。


    举例:

    • 可接受：0⩽x1⩽30 \leqslant x_1 \leqslant 30⩽x1​⩽3 , −1⩽x2⩽0.5-1 \leqslant x_2 \leqslant 0.5−1⩽x2​⩽0.5
    • 不可接受: −100⩽x3⩽100-100 \leqslant x_3 \leqslant 100−100⩽x3​⩽100 , −0.0001⩽x4⩽0.0001-0.0001 \leqslant x_4 \leqslant 0.0001−0.0001⩽x4​⩽0.0001
      吴恩达个人经验认为 −3⩽x⩽3-3 \leqslant x \leqslant 3−3⩽x⩽3 and −13⩽x⩽13-\dfrac{1}{3} \leqslant x \leqslant \dfrac{1}{3}−31​⩽x⩽31​ 是合适的
      (足够接近梯度下降法就可以很好的工作)

  2. Mean normalization 均值标准化

    • Replace xix_ixi​ with xi−μix_i-\mu_ixi​−μi​ to make features have approximately zero mean (Do not apply to x0=1x_0 = 1x0​=1).
      使用 xi−μix_i-\mu_ixi​−μi​ 取代 xix_ixi​ 使训练集均值约为 0。
      (此方法不适用于 x0=1x_0 = 1x0​=1 )


    举例：

    • x1=size−10002000x_1 = \dfrac{size - 1000}{2000}x1​=2000size−1000​ , x2=#bedrooms−25x_2 = \dfrac{\#bedrooms-2}{5}x2​=5#bedrooms−2​
      得到的均值归一化范围为 : −0.5⩽x1⩽0.5 , −0.5⩽x2⩽0.5-0.5 \leqslant x_1 \leqslant 0.5 \ , \ -0.5 \leqslant x_2 \leqslant 0.5−0.5⩽x1​⩽0.5 , −0.5⩽x2​⩽0.5

  3. 综上
    x=x−μs1x = \dfrac{x-\mu}{s_1}x=s1​x−μ​ 此方法是一种接近取值的方法，目的是让梯度下降法可以更高效的运行

    • μ\muμ 是 x 训练集的平均数 (avg value of x in training set)
    • s1s_1s1​ 取值范围 (range : max−minmax-minmax−min ) 或者 标准差 ( standard deviation )

    “调试” 如何确保梯度下降法正确运行

  • How to choose learning rate α\alphaα.

    如何选择合适的学习率 α\alphaα

  1. 确保梯度下降运算正确

    • 观察迭代次数增加的同时 minθJ(θ)\underset{\theta}{min}J(\theta)θmin​J(θ)是否呈下降趋势。


      纵轴 ：minθJ(θ)\underset{\theta}{min}J(\theta)θmin​J(θ)
      横轴 : No. of iterations 迭代次数
      在很多迭代次数后 minθJ(θ)\underset{\theta}{min}J(\theta)θmin​J(θ) 不再变化，表示进入局部最优解，收敛状态

    • Convergence test 收敛测试


      Declare convergence if J(θ)J(\theta)J(θ) decreases by less than 10−310^{-3}10−3 in one iteration.
      如果在下一次迭代中 J(θ)J(\theta)J(θ) 下降 ϵ\epsilonϵ 少于 10−310^{-3}10−3 则认为收敛。

  2. 如何选择合适的学习率

    • 如果 minθJ(θ)\underset{\theta}{min}J(\theta)θmin​J(θ) 随着迭代次数不断 上升 (橘黄色线)，则说明需要减小学习率 α\textcolor{red}{\alpha}α

    • 如果 minθJ(θ)\underset{\theta}{min}J(\theta)θmin​J(θ) 随着迭代次数不断 上升下降反复抖动 (蓝色线) ，则说明需要减小学习率 α\textcolor{red}{\alpha}α

    • 如果学习率 α\alphaα 过小，训练收敛速度会很慢

总结

  • If α\alphaα is too small : slow convergence.

    如果学习率 α\alphaα 太小，收敛会很慢。

  • If α\alphaα is too larget : J(θ)J(\theta)J(θ) may not decrease on every iteration; may not converge.

    如果学习率 α\alphaα 太大，J(θ)J(\theta)J(θ) 在每次迭代不会下降，甚至不会收敛。
    (也有可能出现收敛缓慢的问题)

  • α\alphaα 可以尝试 ...,0.001,0.01,0.1,1,......,0.001,0.01,0.1,1,......,0.001,0.01,0.1,1,... (一般 10 倍增长,可采用二分法逐步找到合适的学习率α\alphaα)

    • x1=(size)x_1 = (size)x1​=(size)

    • x3=(size)3x_3 = (size)^3x3​=(size)3
      则：
      hθ(x)=θ0+θ1x1+θ2x2+θ3x3=θ0+θ1(size)+θ2(size)2+θ3(size)3h_\theta(x)=\theta_0 + \theta_1 x_1 + \theta_2 x_2 + \theta_3 x_3 = \theta_0 + \theta_1 (size) + \theta_2 (size)^2 + \theta_3 (size)^3hθ​(x)=θ0​+θ1​x1​+θ2​x2​+θ3​x3​=θ0​+θ1​(size)+θ2​(size)2+θ3​(size)3
